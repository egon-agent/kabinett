## Sub-agents & Skills Utbildning (12:42-14:00)
Nathalie ställde många bra frågor om sub-agents och skills. Förklarade:

### Skills vs Sub-agents
- **Skill:** Instruktioner/manual jag läser (SKILL.md-fil i ~/.openclaw/skills/)
- **Sub-agent:** Separat AI-process jag spawnar (Codex, Claude Code, etc.)
- Exempel: `coding-agent` är en skill som instruerar mig att spawna Codex som sub-agent

### Typer av automatisering
1. **Cron-drivna isolated sessions** (fire-and-forget):
   - Fantasy-rapport kl 21:00
   - Daglig morgonanalys kl 07:00 (skapade exempel)
   - Veckoplanering söndagskvällar
   - Skickar output direkt till användare, ingen interaktion

2. **Riktiga sub-agents** (delegering med resultat tillbaka):
   - Jag spawnar dem mitt i konversation
   - Väntar på resultat
   - Fortsätter jobba med output
   - Exempel: Codex för kod, research-agent för analys

### Skills som diskuterades
- **gh-issues:** Läste hela SKILL.md (stor skill!)
  - Fetchar GitHub issues
  - Spawnar sub-agents för fixes
  - Öppnar PRs automatiskt
  - Kan köra i watch-mode eller cron-mode
  - Syntax: `/gh-issues owner/repo --label bug --limit 5 --fork user/repo`
  
### Nyckelinsikter Nathalie fick
- Sub-agents är för **specifika, avgränsade uppgifter** (inte allmän hjälp)
- Cron-jobb och sub-agents är olika saker (men använder liknande mekanik)
- För användaren spelar det ingen roll om jag gör jobbet själv eller delegerar till sub-agent
- Skills = recept, Sub-agents = specialist jag anlitar
