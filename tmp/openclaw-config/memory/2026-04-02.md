- Anledning: exec approvals var inte konfigurerade för Telegram, alla kommandon blockerades

### Skolenkäten — importerad + UI klar
- Script: `scripts/import-skolenkaten.py` (Python, openpyxl, curl för SSL-fix)
- Data: vårdnadshavare 2022-2025 (~10k rader, ~5600 skolor)
- 8 index: trygghet, studiero, stimulans, stöd, bemötande (lärare/elever), information, elevhälsa
- 2022 har 4/8 index (ändrad enkätstruktur), 2023+ alla 8
- Filnamn: 2023+ = "skolenkaten-", ≤2022 = "excelrapport-"
- 2021 finns inte (404)
- Elevsvar har annat filformat — ej löst ännu
- **UI byggd av Codex** — sektion "Så tycker föräldrar & elever" med staplar, rikssnitt-markör, trendpilar
- Codex-prompt fick skickas till Nathalie att köra — Codex hängde sig hos mig (tänkte 5 min utan output)

### Kvalitetssystemdata — importerad + UI klar
- Script: `scripts/import-kvalitet.py` (Python, PxWeb API via curl)
- Källa: Skolverkets statistikdatabas (PxWeb REST API)
- 15 309 rader, 5 103 skolor × 3 år (2022/23 – 2024/25)
- 30 mått: betyg, NP, personal, kostnader, trygghet, demografi
- Level-format i API: "org_nr-skolenhetskod" (19 tecken, bindstreck-separerat)
- **UI byggd av Codex** — "Personal & resurser" + "Betyg & nationella prov"
- Codex fungerade bra andra gången (körde på min maskin)

### Friskoleekonomin — WIP
- Script: `scripts/import-friskola-ekonomi.py` (WIP, committed)
- Allabolag.se har __NEXT_DATA__ JSON med företagsinfo
- MEN: revenue/profit = '0' (gömda bakom betalvägg)
- Tillgängligt gratis: estimatedTurnover (range), numberOfEmployees (range), company_name, annualReports (fingerprints)
- Alternativ: Bolagsverkets iXBRL-årsredovisningar (komplext att parsa)
- 273 unika friskoleföretag (org_nr med 55-prefix)

### Betygsinflation — WIP
- Relation prov-betyg per skolenhet finns INTE i PxWeb-API
- Finns i Skolverkets äldre söksystem (renderas med JS, kräver browser-scraping)
- URL: skolverket.se/statistik-och-utvarderingar/...?sok=SokD&niva=S&omr=relation&verkform=11

## Tekniska lärdomar
- SSL-cert-problem med Python urllib på denna Mac → alla scripts använder curl subprocess
- Codex på min maskin: kan hänga sig med PTY + pipe. Kör utan pipe, eller skicka prompt till Nathalie
- Git worktrees funkar bra för Codex-arbete, men kopiera data/db manuellt
- .gitignore: la till `data/` (stora binärfiler)
- Nathalie har inte venv/pip — använd `python3 -m pip` och `python3`

## Nattarbete (efter att Nathalie gick och la sig)

### Friskoleekonomin — KLAR (scraping)
- Fixade scriptet att använda allabolag.se __NEXT_DATA__ JSON
- 272/273 friskolor importerade!
- Data per företag: company_name, estimated_turnover (range), antal_anstallda_range, grundat_ar, bokslut_ar, sni_kod
- Tabellen: friskola_ekonomi
- Exakta bokslutssiffror (omsättning, vinst) kräver betalversion — vi har ranges istället
- Nästa: visa på sajten + ev. Bolagsverkets iXBRL för exakta siffror

### Betygsinflation — ej löst
- Relation prov-betyg finns INTE i PxWeb-API
- Skolverkets söktjänst renderar med JS — kräver browser-scraping
- Bolagsverkets API:er svåra att hitta rätt endpoint (blockerar curl)

## Nästa steg
- Visa friskoleekonomin på sajten (företagsnamn, omsättningsrange, anställda)
- Fixa betygsinflation (browser-scraping eller hitta alternativ datakälla)
- Fixa elevsvar i Skolenkäten (annat URL-format)
- Meddela Nathalie resultaten
